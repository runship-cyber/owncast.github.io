    Owncast v0.0.2

    September 27, 2020

    v0.0.2 has a bunch of updates, mostly focused on the web frontend.

    Major changes #

    • Rewrite of the web interface with Preact, removing Vue. [#103]
    • Add support for embedding chat externally. [#96]
    • Add support for embedding the video player externally. [#110]
    • Your username will get highlighted in chat when mentioned. [#100]
    • Emoji picker w/ custom emoji support. [#86]
    • Autocomplete usernames in composing chat messages by hitting tab. [#99]
    • Improve support for autolinking, and embedding YouTube or Instagram links in chat. [#93]
    • Server alerts to the console when there is excessive hardware utilization. [#115]
    • Updated version of Video.js and associated http-streaming library. [#133]
    • Improve max bitrate compatibility with HLS spec. [#106]
    • Stop jumping to bottom of chat when scrolling and reading previous messages. [#101]
    • Username colors are tweaked to be easier to read. [#170]
    • Improved mobile layout in the web UI. [#140]
    • Remove Cache-Control header when sending to external S3 storage to support Digital Ocean Spaces. [#117]
    • Stream gets cleaned up after 5 minutes of being disconnected. [#152]

    Visit the documentation to read more about how to take advantage of embedding and custom emoji.

    You can get the release from GitHub or on Docker Hub.

    Upgrading from v0.0.1 #

    This release has a rewrite of the web frontend, so any customization you made previously will need to be re-applied to this new web app. The simplest way to upgrade is just overwrite the old version with the new version, but it won’t delete any old files that are no longer used.

    However, you’re free to upgrade any way you like, but make sure your chat.db, config.yaml, webroot/static/content.md and stats.json files are saved.

    1. Stop the service from running.
    2. Move the zip file of 0.0.2 to your previous install location.
    3. Unzip the file, allowing it to overwrite old files.
    4. Restart the service.
